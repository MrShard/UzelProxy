# UzelProxy

Браузерное расширение (Manifest V3) для обхода блокировок сайтов. Личный форк FastProxy — без серверной инфраструктуры оригинала, с переработанной логикой и дизайном под собственное видение.

## Зачем этот проект

Расширение **FastProxy** больше недоступно в магазинах расширений и застряло в развитии: устаревший дизайн,的功能ность и архитектура перестали отвечать текущим требованиям, а поддержка прекращена.

UzelProxy — это переосмысление FastProxy «под себя». Проект решает ровно те проблемы, которые интересны лично мне, меняет оформление под собственное видение и нацелен на создание расширения, удобного мне и моему окружению. Это не попытка клонировать оригинал, а самостоятельная ветка развития на его базе.

## Чем отличается от оригинала

- **Убран встроенный серверный прокси.** Оригинал работал через собственные серверы разработчика (скрытые за динамическим конфигом). UzelProxy полагается только на **собственный прокси пользователя** (HTTP/S, SOCKS5, TOR, WARP) — никакой чужой инфраструктуры.
- **PAC-скрипт строится на клиенте**, а не скачивается готовым с сервера. Логика маршрутизации прозрачна и полностью в коде расширения.
- **Готовый список исключений из этого репозитория.** Галочка «Использовать готовый список исключений» подтягивает [`blocklist.txt`](./blocklist.txt) (19 729 доменов) через CDN и обновляется автоматически раз в неделю. Список открыт — его можно дополнять.
- Вырезаны трекинг, удалённые «действия» на сайтах и kill-switch оригинала.

## Как это работает

Расширение генерирует PAC-скрипт (Proxy Auto-Config) и применяет его через `chrome.proxy.settings`. Для каждого сайта PAC решает: направить через ваш прокси или напрямую. Приоритет (первое совпадение выигрывает):

1. Готовый список исключений из git → **напрямую**
2. Пользовательские исключения → **напрямую**
3. «Проксировать только домены из списка» → прокси только для списка
4. «Добавить домены для проксирования» → аддитивно
5. «Проксировать весь трафик» → весь трафик через прокси (режим VPN)
6. По умолчанию → **напрямую** (без своего прокси расширение пассивно)

> **Важно:** без собственного прокси расширение ничего не проксирует — список исключений имеет смысл только поверх включённого прокси пользователя. Это следствие отказа от встроенного серверного прокси.

## Возможности

- Собственный прокси: HTTP/S и SOCKS5 (включая TOR и WARP как локальный прокси)
- Готовый список исключений из git (автообновление раз в неделю)
- Собственные списки доменов: не проксировать / только проксировать / добавить / весь трафик
- Поддержка wildcard (`*.domain.com`)
- Отключение других прокси/VPN-расширений при конфликте
- Индикация проксируемых сайтов на иконке

## Список исключений

Файл [`blocklist.txt`](./blocklist.txt) — открытый список доменов, которые открываются напрямую, минуя ваш прокси. Формат: один домен на строку, `#` — комментарий, `*` — wildcard. Изменения, запушенные сюда, становятся доступны пользователям в течение нескольких дней через CDN.

## Установка (в разработке)

1. Клонируйте репозиторий.
2. В Chrome/Edge: `chrome://extensions` → включите **Режим разработчика** → **Загрузить распакованное расширение** → выберите папку проекта.
3. Откройте расширение, введите свой прокси, при желании включите «Готовый список исключений».
